# University Relational Database – Design & Query

A normalized relational database that simulates a university information
system storing students, instructors, courses and enrollments.

## Project Structure

```
checkpoint_university_db/
├── 01_schema.sql        -- DDL: tables, keys, constraints
├── 02_insert_data.sql   -- Sample data (students, instructors, courses, enrollments)
├── 03_queries.sql       -- The 5 required queries (SELECT / UPDATE / DELETE)
└── README.md
```

## Entity-Relationship Overview

```
Instructors ──< Courses ──< Enrollments >── Students
   1:N            1:N          M:N
```

- One **Instructor** teaches many **Courses**.
- One **Course** can have many **Students** enrolled, and one **Student**
  can take many **Courses** → resolved via the **Enrollments** junction
  table.

## Schema

### Students
| Column      | Type         | Constraints                          |
|-------------|--------------|--------------------------------------|
| student_id  | INT          | PRIMARY KEY, AUTO_INCREMENT          |
| name        | VARCHAR(100) | NOT NULL                             |
| email       | VARCHAR(150) | NOT NULL, UNIQUE                     |
| age         | INT          | NOT NULL, CHECK (age > 17)           |

### Instructors
| Column        | Type         | Constraints                  |
|---------------|--------------|------------------------------|
| instructor_id | INT          | PRIMARY KEY, AUTO_INCREMENT  |
| name          | VARCHAR(100) | NOT NULL                     |
| department    | VARCHAR(100) | NOT NULL                     |

### Courses
| Column        | Type         | Constraints                                  |
|---------------|--------------|----------------------------------------------|
| course_id     | INT          | PRIMARY KEY, AUTO_INCREMENT                  |
| title         | VARCHAR(150) | NOT NULL, UNIQUE                             |
| credits       | INT          | NOT NULL, CHECK (credits BETWEEN 1 AND 6)    |
| instructor_id | INT          | NOT NULL, FK → Instructors(instructor_id)    |

### Enrollments
| Column          | Type        | Constraints                                          |
|-----------------|-------------|------------------------------------------------------|
| student_id      | INT         | PK (composite), FK → Students(student_id) CASCADE    |
| course_id       | INT         | PK (composite), FK → Courses(course_id)   CASCADE    |
| grade           | VARCHAR(2)  | CHECK (grade IN ('A','B','C','D','E','F') OR NULL)   |
| enrollment_date | DATE        | NOT NULL, DEFAULT CURRENT_DATE                       |

## Normalization (up to 3NF)

- **1NF** – Every column holds a single atomic value; no repeating groups.
- **2NF** – No partial dependency. The only composite key is in
  `Enrollments` and every non-key column (`grade`, `enrollment_date`)
  depends on the full key `(student_id, course_id)`.
- **3NF** – No transitive dependencies. Instructor attributes (`name`,
  `department`) are stored *only* in `Instructors`; `Courses` references
  the instructor through `instructor_id` instead of duplicating data.

## Required Queries (see `03_queries.sql`)

1. Retrieve all students enrolled in the course **"Database Systems"**.
2. List all courses along with the names of their instructors.
3. Find students who are not enrolled in any course.
4. Update the email address of a student.
5. Delete a course by its ID.

## How to Run

The scripts use standard SQL and were written with **MySQL 8 / MariaDB**
in mind (`AUTO_INCREMENT`, `DEFAULT (CURRENT_DATE)`). To run:

```bash
mysql -u <user> -p <database> < 01_schema.sql
mysql -u <user> -p <database> < 02_insert_data.sql
mysql -u <user> -p <database> < 03_queries.sql
```

For **PostgreSQL**, replace `INT AUTO_INCREMENT` with `SERIAL` (or
`INT GENERATED BY DEFAULT AS IDENTITY`) and `DEFAULT (CURRENT_DATE)`
with `DEFAULT CURRENT_DATE`.

For **SQLite**, replace `INT AUTO_INCREMENT` with
`INTEGER PRIMARY KEY AUTOINCREMENT` (and remove the inline `PRIMARY KEY`
line accordingly).
